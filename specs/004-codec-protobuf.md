# codec-protobuf: Protobuf-backed CodecFactory

## What to build

A Maven module providing a Protocol Buffers-backed `CodecFactory` implementation.
Supports any `GeneratedMessageV3` (protobuf-java generated message class).

### Module: codec-protobuf

Package: `org.jwcarman.codec.protobuf`

**ProtobufCodecFactory** implements `CodecFactory`:
- `create(Class<T>)`: verifies `T` extends `GeneratedMessageV3`, then returns a
  `ProtobufCodec<T>` that uses the class's static `parseFrom(byte[])` method
  (accessed via reflection once, cached)
- `create(TypeRef<T>)`: extracts the raw class from `TypeRef.getType()` and delegates
  to `create(Class<T>)`. Protobuf doesn't use generics in its message types, so
  `TypeRef` is supported but rarely needed.
- Throws `IllegalArgumentException` if the type is not a `GeneratedMessageV3` subclass

**ProtobufCodec<T>** implements `Codec<T>`:
- `encode`: `((GeneratedMessageV3) value).toByteArray()`
- `decode`: invoke the cached `parseFrom(byte[])` static method
- `type`: returns the message class
- Thread-safe (protobuf serialization is stateless)
- Caches the `parseFrom` `Method` reference per type for performance

**ProtobufCodecAutoConfiguration**:
- `@AutoConfiguration(before = CodecAutoConfiguration.class)`
- `@ConditionalOnClass(GeneratedMessageV3.class)`
- `@ConditionalOnMissingBean(CodecFactory.class)`
- Creates `ProtobufCodecFactory` bean

## Acceptance criteria

- [ ] Module compiles and produces a jar
- [ ] `AutoConfiguration.imports` registration exists
- [ ] Auto-config creates `ProtobufCodecFactory` when protobuf-java is on classpath
  and no other `CodecFactory` exists

## Implementation notes

- Protobuf messages always have a static `parseFrom(byte[])` method generated by
  protoc. We find it once via `type.getMethod("parseFrom", byte[].class)` and cache it.
- For tests, create a simple `.proto` file in `src/test/proto/` and use the
  `protobuf-maven-plugin` to generate test message classes. A simple message with a
  few fields (string, int, nested message) is sufficient.
- Unlike Jackson/Gson, Protobuf doesn't do generic types — each message is a concrete
  class. The `TypeRef` path just extracts the raw class and delegates.
- Dependency: `com.google.protobuf:protobuf-java`
